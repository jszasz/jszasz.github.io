Job Description

In my role as a Junior Quality Assurance Analyst, I assisted with testing of major projects that all had to do with client facing documents. Whether that was a physical printed document, or a digital PDF. I also did some web testing for a new client facing website that allows clients to view their insurance information online.

Here is a list of my official duties:

  • Assisting with system testing
  • Providing input to the development of acceptance test plans
  • Assisting in the development and maintenance of test case documentation
  • Documenting problems encountered in testing
  • Participating in verifications of implementations
  • Participating in walkthroughs of testing documentation

I was at first only assisting with these tasks on behalf of the other testers on the team, but after I was fully onboarded into the roll, I was doing the exact same tasks as the other contract testers, and eventually in charge of my own test project from start to finish.

Tasks / Roles

The tasks that I was assigned to changed over the eight months as projects were finished and I was moved to other work. This was intentional by my manager as it allowed me to gain experience doing other tasks that weren't necessilary part of my job description. I will list my tasks / roles in chronological order:

Farm Project Testing

There was a project called “The Farm Project” that was ongoing when I started working. The Farm policy system was being moved from an older system to a newer one, and this required testing of everything. PDF outputs from the old system were compared to PDF outputs from the new system, and all the data had to be the same on both.

This was where I started learning, so I would assist the other testers and ask them questions as I was working. I needed to learn the insurance business as well as our insurance system called Policy Centre. The other testers would handle everything else. They would look at business documents and create test requirements and policies that covered everything.

I would mainly be testing an old PDF vs a new PDF and logging any differences as issues for the programmers to look at. I would then check off that I tested that policy in an Excel spreadsheet that another tester had made. I would also create some test policies in Policy Centre based on another tester’s requirements, or create test clients in our database for these test policies.

I was helpful on this project as I was taking some work from the other testers, and it gave me a chance to learn how everything works without being responsible for a timeline.

Selenium Farm Policy Automation

This was a personal project I started while working on the Farm Project. My co-workers wanted me to improve on the previous co-op student’s work and try to make an application to automate policy creation in the Policy Centre application (which is just a web application).

Workflow

The program allows a non technical tester to fill out data in an excel spreadsheet (or copy and rearrange existing data), and run the program without changing any code. The program will automate the entire policy creation process for the tester, based on the data they entered in the Excel document. It will log in with the supplied credentials, and make a policy with the supplied data. The tester can then edit the policy if there is anything specific that they didn’t include in the Excel file, and then they can submit the policy. They can then use the PDF for testing. This allows the tester to focus on other things rather than making policies that are only slightly different, from scratch. They can make multiple of the same policies and then change what is needed to be different.
