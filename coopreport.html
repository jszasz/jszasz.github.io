Selenium Farm Policy Automation

This was a personal project I started while working on the Farm Project. My co-workers wanted me to improve on the previous co-op student’s work and try to make an application to automate policy creation in the Policy Centre application (which is just a web application).

Workflow

The program allows a non technical tester to fill out data in an excel spreadsheet (or copy and rearrange existing data), and run the program without changing any code. The program will automate the entire policy creation process for the tester, based on the data they entered in the Excel document. It will log in with the supplied credentials, and make a policy with the supplied data. The tester can then edit the policy if there is anything specific that they didn’t include in the Excel file, and then they can submit the policy. They can then use the PDF for testing. This allows the tester to focus on other things rather than making policies that are only slightly different, from scratch. They can make multiple of the same policies and then change what is needed to be different.


Sample of what the Excel file could look like.

Technical

The program was made in Java, using two main JAR’s. One is called Selenium Web Driver, which runs the web browser. The other is Apache POI, which is a Java API for Microsoft Office documents. This allows the program to access the data in the Excel file.

The program loads the data from the Excel file, and using that as its input, will start clicking and typing on the webpage. For example, an input of “100100100” might be a sequence of radio buttons, with “1” meaning yes and “0” meaning no, and the program will click on those options. Another input might be “2” which would relate to the second option in a select box. A third example might be “04/05/1980” which will be typed into a birthday input box. The program knows which data is needed depending on previous options chosen, it is basically a very complex state machine or a large set of “if” statements.

Release 36.5 Messagepoint Upgrade Testing

This project was smaller in comparison to other test projects, but it was different as I was the lead tester on the project. The project involved upgrading one of our vendor products that takes our XML data files from our system and turns them into PDF’s. Since I was the test lead, I was responsible for creating all of the test planning documents and weekly status updates. I was exposed to a lot more project management processes than I was used to.

It took two weeks of testing from start to finish during the upgrade, and most of it was very repetitive. I would run the XML data files in the current version of the software, then run the same file in the new version of the software. After that I would need to compare for any differences. After that phase, I re-ran all the data files in the upgraded system in an “End to End” test. Meaning it was connected to our entire system, not just the vendor’s software.

It was very taxing on me to be the test lead of this project. I was constantly trying to estimate how much work I had done in what period of time, and use that estimate to see how much work I still had to do. I used this to update the Project Manager on my progress during meetings. I was always trying to work as fast as I could just to get ahead.

It was nice to get some independance, and act more like a normal tester rather than a co-op student.

Learning Goals

I had three learning goals for the eight month work term that I wanted to improve on to help me in my future career. These goals can roughly apply to any co-op student, and should be taken into consideration to gain the most from a co-op term. Here is a summary of the goals that I improved upon the most, and will continue to improve upon before I graduate:

Goal 1 - Increase Programming Experience

I wanted to improve upon my programming experience, and have at least one thing I could add to my resume that was programming related. I did get my hands on a few technologies at Co-operators, as well as working in a Windows based workspace (where as usually we work in UNIX based systems in school). This goal was hard to accomplish fully as outside of work I was unmotivated to work on some personal coding projects, but I did still get some resume experience with my Farm Policy Automation program I made. I will have to continue working on this goal before I graduate, to help improve upon my resume and portfolio.

Goal 2 - Better Network Myself To Future Employers

I wanted to learn as much as I could about networking, and help prepare myself for my future full time job search. I helped my manager with interviews which gave me a different perspective on the hiring process, which gave me an insight into what is important to include on my resume. I also created a Linkedin profile and added all of my work experience to it. I added most of my current co-workers, and a good chunk of previous co-workers. Now that my resume is online, I am making a online website so that I also have a web presence with links to some projects. I will need to improve upon my portfolio before I graduate, and constantly update my website and linkedin profile. These are great things to have for potential employers to see.

Goal 3 - Learn Insurance Business and Professional Skills

I can personally feel the benefits from the first part of this learning goal as I am now fluent in insurance terms and the reasoning behind why certain things make insurance cheaper or more expensive. I have personally been with The Co-operators my entire life, and my advisor knows I did a co-op with the company. I feel more confident that I can read my insurance documents and that I understand what they mean. I also look forward to future conversations with my insurance advisor where we are both speaking the same language.

I also gained experience with project planning aspects of my job as I had to understand the projects that I was working on. I also had the chance to make two project testing documents detailing the testing effort for the project. The first project went on hold till next year, but the second project I was the lead tester for, so I experienced the project planning aspects first hand as I made the planning documents, presented them, carried out the testing, and made timeliness and sent out updates on progress and detailed any issues found.

It was very enriching to be in charge of my own project in the later half of my co-op term, as I feel it helped my organization skills and my time management skills, as everything that happened on the project would come back to me if something was wrong. I believe I successfully improved on this learning goal.
